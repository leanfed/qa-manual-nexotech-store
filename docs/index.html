Requerimientos principales

Funcionales y no funcionales

Funcionales

  • Visualizar productos tecnológicos en la página principal.
  • Buscar productos por nombre o descripción.
  • Filtrar productos por categoría.
  • Consultar detalle de producto.
  • Agregar, modificar, eliminar y vaciar productos del carrito.
  • Conservar datos del carrito mediante LocalStorage.
  • Acceder al panel administrador y gestionar productos.

No funcionales

  • Diseño responsive adaptable a vista mobile.
  • Claridad visual en botones, cards y formularios.
  • Carga adecuada de recursos en GitHub Pages.
  • Recuperación correcta de datos almacenados localmente.

Funcionales, interfaz, responsive, persistencia, negativas y regresión básica.

Observaciones registradas

Limitaciones conocidas
Baja

OBS-001 · Autenticación administradora simulada

La autenticación del panel administrador se realiza del lado del cliente y no valida contra backend.

Media

OBS-002 · Checkout sin generación de orden

El flujo de carrito no genera una orden de compra ni integra un checkout real.

Media

OBS-003 · Administración limitada a LocalStorage

Los cambios del catálogo quedan asociados al navegador local y no se comparten entre usuarios.
